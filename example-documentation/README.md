# Example documentation

A worked example of the **final documentation** deliverable — it shows the
expected structure and level of detail for the term project. The domain here is
a fictional *Club Library*; your own documentation describes your own
application.

Build it: from the repository root run `make` → the PDF lands in
`out/documentation.pdf`.

Your real documentation is submitted as its **own GitHub repository** with an
integrated LaTeX build workflow and `Makefile`. Copy the root `Makefile` and
`../.github/workflows/build.yml` into that repository so every tag push builds
your PDF and attaches it to a GitHub Release.

Reused code (lectures / earlier exercises) is cited in the *Sources and reuse*
section — do the same in yours.
